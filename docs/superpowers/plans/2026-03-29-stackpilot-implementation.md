# Stackpilot Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the Stackpilot framework — a set of agent definitions, git hooks, and scripts that let Claude Code autonomously execute projects from design doc to production-ready state.

**Architecture:** Five specialized agents (PM, Dev, QA, Architect, Docs) are defined globally in `~/.claude/agents/`. A Coordinator agent is triggered by git hooks (`post-checkout`, `post-commit`) via `claude -p` headless mode. Project state lives in `tasks/` YAML files. A `stackpilot init` shell script bootstraps any new project.

**Tech Stack:** Bash (hooks + init script), Markdown (agent definitions + skills), YAML (task state files), `claude` CLI (headless agent execution), `osascript` (macOS desktop notifications)

You are the PM Agent for Stackpilot. Your sole job is to read an implementation plan and produce a structured task backlog.

## Trigger Context

You are called when a new `docs/superpowers/plans/*.md` or `docs/specs/*.md` file is committed. The calling hook passes the new file path as context.

## Your Process

1. Read the implementation plan file passed to you
2. Read `tasks/backlog.yml` if it exists (to avoid duplicate task IDs)
3. Decompose the plan into atomic tasks — one task per logical unit of work
4. Write the full task list to `tasks/backlog.yml`

## Task Decomposition Rules

- Each task must be completable by a single agent in one session
- Tasks of type `dev` are followed by a `qa` task for the same component (create both)
- Assign `depends_on` accurately — a `qa` task depends on its `dev` task
- Never create a task without a clear, self-contained description
- Estimate priority: tasks that unblock others are `high`, standalone features are `medium`, polish/docs are `low`

## Output Format

Write `tasks/backlog.yml` with this exact structure:

```yaml
- id: TASK-001
  title: <imperative title>
  type: dev          # dev | qa | docs | arch
  priority: high     # high | medium | low
  status: pending
  depends_on: []
  description: |
    <3-5 sentences: what to build, where (exact file paths), and how to verify it works>
  assigned_to: null
```

## Constraints

- Do NOT modify any source code
- Do NOT create tasks outside `tasks/backlog.yml`
- If `tasks/backlog.yml` already has tasks, append new ones — do not overwrite
- Task IDs must be sequential and not conflict with existing IDs
```

- [ ] **Step 1: Create Coordinator agent definition**

Create `~/.claude/agents/coordinator-agent.md`:

```markdown
---
name: coordinator-agent
description: Orchestrates the Stackpilot sprint. Reads task state, processes user replies in NEEDS_REVIEW.md, dispatches specialist agents, and handles timeouts. Runs headless via git hooks.
tools: Read, Write, Bash, Glob
---

You are the Stackpilot Coordinator. You run automatically on git events.

## Entry Checklist (run in this order every time)

### 1. Process NEEDS_REVIEW.md

Read `tasks/NEEDS_REVIEW.md`:
- If file is empty or does not exist: continue to step 2
- If file contains a `REPLY:` line:
  - Parse the reply
  - Find the task ID in the escalation header (e.g., `[DEV][TASK-003]`)
  - Update that task in `tasks/backlog.yml`: set `status: pending` (unblock it)
  - Overwrite `tasks/NEEDS_REVIEW.md` with empty content
  - Continue to step 2
- If file has content but NO `REPLY:` line:
  - Send desktop notification: `osascript -e 'display notification "Stackpilot needs your input" with title "NEEDS_REVIEW.md has an open question"'`
  - Stop — do not dispatch any new tasks until the review is resolved

### 2. Check for timed-out tasks

Read `tasks/in-progress.yml`. For each task:
- Check `started_at` timestamp
- If current time - `started_at` > `coordinator.timeout_hours` from `stackpilot.config.yml`:
  - Set task `status: failed` in `tasks/backlog.yml`
  - Append to `tasks/NEEDS_REVIEW.md`:
    ```
    [COORDINATOR][TASK-ID] Task timed out after X hours with no status update
    Option A: Re-queue the task (set back to pending)
    Option B: Skip this task and continue
    Recommendation: Option A, review agent logs first
    ```
  - Send desktop notification: `osascript -e 'display notification "Task TASK-ID timed out" with title "Stackpilot: action needed"'`
  - Remove from `tasks/in-progress.yml`

### 3. Dispatch pending tasks

Read `tasks/backlog.yml`:
- Find all tasks with `status: pending`
- For each pending task, check if all `depends_on` task IDs have `status: done`
- Take up to `coordinator.worktree_limit` tasks that are ready
- For each selected task:
  - Set `status: in-progress`, `assigned_to: <agent-type>-agent`, `started_at: <ISO timestamp>`
  - Update `tasks/in-progress.yml`
  - Dispatch the appropriate agent (see dispatch rules below)

### 4. Dispatch rules

| Task type | Agent to dispatch | How |
|-----------|------------------|-----|
| `arch` | architect-agent | `claude -p "You are the architect-agent. Review task TASK-ID." --allowedTools Read,Glob,Grep,WebSearch` |
| `dev` | dev-agent | First run architect-agent if no arch review exists, then: `claude -p "You are the dev-agent. Implement task TASK-ID." --allowedTools Read,Edit,Write,Bash,Glob,Grep` |
| `qa` | qa-agent | `claude -p "You are the qa-agent. Test task TASK-ID." --allowedTools Read,Write,Bash,Glob,Grep` |
| `docs` | docs-agent | `claude -p "You are the docs-agent. Document task TASK-ID." --allowedTools Read,Edit,Write,Glob` |

### 5. Check for sprint completion

If `tasks/backlog.yml` has zero tasks with `status` of `pending` or `in-progress`:
- Send completion notification:
  ```bash
  osascript -e 'display notification "All tasks complete — ready for review" with title "Stackpilot: Sprint Done 🎉"'
  ```
```

- [ ] **Step 1: Create post-checkout hook**

Create `scripts/hooks/post-checkout.sh`:

```bash
#!/usr/bin/env bash
# Stackpilot post-checkout hook
# Runs Coordinator when switching branches — natural entry point into project work

set -euo pipefail

REPO_ROOT="$(git rev-parse --show-toplevel)"

# Only run if stackpilot is initialized (tasks/ directory exists)
if [ ! -d "$REPO_ROOT/tasks" ]; then
  exit 0
fi

# Only run if there are tasks to process
if [ ! -f "$REPO_ROOT/tasks/backlog.yml" ]; then
  exit 0
fi

echo "[stackpilot] Running Coordinator..."
cd "$REPO_ROOT"
claude -p "$(cat ~/.claude/skills/stackpilot/coordinator.md)" \
  --allowedTools "Read,Write,Bash,Glob" \
  --project-dir "$REPO_ROOT" &

# Run in background so git checkout completes immediately
echo "[stackpilot] Coordinator dispatched (running in background)"
```

- [ ] **Step 2: Create post-commit hook**

Create `scripts/hooks/post-commit.sh`:

```bash
#!/usr/bin/env bash
# Stackpilot post-commit hook
# Detects new spec/plan files and triggers PM Agent to populate backlog

set -euo pipefail

REPO_ROOT="$(git rev-parse --show-toplevel)"

# Only run if stackpilot is initialized
if [ ! -d "$REPO_ROOT/tasks" ]; then
  exit 0
fi

# Detect if this commit added a new plan or spec file
NEW_SPECS=$(git diff HEAD^ HEAD --name-only 2>/dev/null | grep -E "^docs/(specs|superpowers/plans)/.*\.md$" || true)

if [ -z "$NEW_SPECS" ]; then
  exit 0
fi

echo "[stackpilot] New spec detected: $NEW_SPECS"
echo "[stackpilot] Running PM Agent to populate backlog..."

cd "$REPO_ROOT"
claude -p "You are the pm-agent. A new implementation plan was just committed. File(s): $NEW_SPECS. Read the plan and populate tasks/backlog.yml." \
  --allowedTools "Read,Write,Glob" \
  --project-dir "$REPO_ROOT" &

echo "[stackpilot] PM Agent dispatched (running in background)"
```

- [ ] **Step 1: Create update script**

Create `scripts/update-gstack.sh`:

```bash
#!/usr/bin/env bash
# Stackpilot gstack updater
# Run weekly (Monday 3:00 AM via cron)
# Pulls gstack, verifies core skills exist, rolls back if broken

set -euo pipefail

GSTACK_DIR="$HOME/.claude/skills/gstack"
REQUIRED_SKILLS=(
  "plan-eng-review"
  "qa"
  "ship"
  "review"
  "investigate"
)

log() { echo "[stackpilot/update-gstack] $*"; }
notify() {
  osascript -e "display notification \"$1\" with title \"Stackpilot: gstack update\"" 2>/dev/null || true
}

# 1. Check gstack is installed
if [ ! -d "$GSTACK_DIR/.git" ]; then
  log "gstack not found at $GSTACK_DIR — skipping update"
  exit 0
fi

# 2. Save current commit for rollback
PREV_COMMIT=$(git -C "$GSTACK_DIR" rev-parse HEAD)
log "Current gstack commit: $PREV_COMMIT"

# 3. Pull
log "Pulling latest gstack..."
if ! git -C "$GSTACK_DIR" pull --ff-only 2>&1; then
  log "Pull failed (possible conflict) — no changes made"
  notify "Pull failed — no changes made. Check $GSTACK_DIR manually."
  exit 1
fi

NEW_COMMIT=$(git -C "$GSTACK_DIR" rev-parse HEAD)

if [ "$PREV_COMMIT" = "$NEW_COMMIT" ]; then
  log "Already up to date — no changes"
  exit 0
fi

log "Updated to $NEW_COMMIT"

# 4. Verify required skill files exist
MISSING=()
for skill in "${REQUIRED_SKILLS[@]}"; do
  # Skills can be in skills/ subdirectory or root — search both
  if ! find "$GSTACK_DIR" -name "${skill}.md" -o -name "${skill}" | grep -q .; then
    MISSING+=("$skill")
  fi
done

if [ ${#MISSING[@]} -gt 0 ]; then
  log "ERROR: Missing skills after update: ${MISSING[*]}"
  log "Rolling back to $PREV_COMMIT..."
  git -C "$GSTACK_DIR" reset --hard "$PREV_COMMIT"
  notify "Update rolled back — missing skills: ${MISSING[*]}. Manual review needed."
  exit 1
fi

log "All required skills verified ✓"
log "gstack updated successfully: $PREV_COMMIT → $NEW_COMMIT"
# Silent on success — no notification needed
```

#!/usr/bin/env bash
# stackpilot init
# Bootstraps a project to use Stackpilot
# Run from the project root: bash ~/Documents/github/stackpilot/scripts/init.sh

set -euo pipefail

STACKPILOT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
REPO_ROOT="$(git rev-parse --show-toplevel 2>/dev/null || pwd)"

log() { echo "[stackpilot init] $*"; }
success() { echo "[stackpilot init] ✓ $*"; }

log "Initializing Stackpilot in: $REPO_ROOT"

# 1. Check gstack is installed
if [ ! -d "$HOME/.claude/skills/gstack" ]; then
  log "Installing gstack..."
  git clone https://github.com/garrytan/gstack "$HOME/.claude/skills/gstack"
  success "gstack installed"
else
  success "gstack already installed"
fi

# 2. Create tasks/ directory structure
mkdir -p "$REPO_ROOT/tasks/done"
mkdir -p "$REPO_ROOT/tasks/arch-review"

if [ ! -f "$REPO_ROOT/tasks/backlog.yml" ]; then
  cp "$STACKPILOT_DIR/templates/backlog.yml" "$REPO_ROOT/tasks/backlog.yml"
  success "Created tasks/backlog.yml"
fi

if [ ! -f "$REPO_ROOT/tasks/NEEDS_REVIEW.md" ]; then
  cp "$STACKPILOT_DIR/templates/NEEDS_REVIEW.md" "$REPO_ROOT/tasks/NEEDS_REVIEW.md"
  success "Created tasks/NEEDS_REVIEW.md"
fi

# Create empty in-progress.yml
if [ ! -f "$REPO_ROOT/tasks/in-progress.yml" ]; then
  echo "# Tasks currently being executed by Stackpilot agents" > "$REPO_ROOT/tasks/in-progress.yml"
  echo "# Do not edit manually" >> "$REPO_ROOT/tasks/in-progress.yml"
  success "Created tasks/in-progress.yml"
fi

# 3. Create stackpilot.config.yml if not present
if [ ! -f "$REPO_ROOT/stackpilot.config.yml" ]; then
  cp "$STACKPILOT_DIR/templates/stackpilot.config.yml" "$REPO_ROOT/stackpilot.config.yml"
  success "Created stackpilot.config.yml (edit to match your project)"
fi

# 4. Install git hooks
HOOKS_DIR="$REPO_ROOT/.git/hooks"

install_hook() {
  local hook_name="$1"
  local source="$STACKPILOT_DIR/scripts/hooks/${hook_name}.sh"
  local target="$HOOKS_DIR/$hook_name"

  if [ -f "$target" ]; then
    # Append stackpilot call to existing hook
    if ! grep -q "stackpilot" "$target"; then
      echo "" >> "$target"
      echo "# Stackpilot" >> "$target"
      cat "$source" >> "$target"
      log "Appended to existing $hook_name hook"
    else
      log "$hook_name hook already has Stackpilot — skipping"
    fi
  else
    cp "$source" "$target"
    chmod +x "$target"
    success "Installed $hook_name hook"
  fi
}

install_hook "post-checkout"
install_hook "post-commit"

# 5. Add tasks/ to .gitignore if not already there
if [ -f "$REPO_ROOT/.gitignore" ]; then
  if ! grep -q "^tasks/done" "$REPO_ROOT/.gitignore"; then
    echo "" >> "$REPO_ROOT/.gitignore"
    echo "# Stackpilot runtime files (keep backlog.yml and NEEDS_REVIEW.md tracked)" >> "$REPO_ROOT/.gitignore"
    echo "tasks/done/" >> "$REPO_ROOT/.gitignore"
    echo "tasks/arch-review/" >> "$REPO_ROOT/.gitignore"
    echo "tasks/in-progress.yml" >> "$REPO_ROOT/.gitignore"
    success "Updated .gitignore"
  fi
fi

echo ""
echo "✅ Stackpilot initialized!"
echo ""
echo "Next steps:"
echo "  1. Edit stackpilot.config.yml (set your test command and coverage threshold)"
echo "  2. Run /brainstorm to plan your first feature"
echo "  3. Commit the design doc — PM Agent will populate tasks/backlog.yml automatically"
echo "  4. Switch branches (git checkout) to trigger the Coordinator"
```

# Stackpilot

Autonomous AI development team. Powered by [gstack](https://github.com/garrytan/gstack) + Claude Code.

## What it does

You give an idea → discuss until design doc → Stackpilot executes to production-ready.

- **PM Agent** decomposes plans into tasks
- **Architect Agent** reviews technical decisions
- **Dev Agent** writes code
- **QA Agent** writes and runs tests
- **Docs Agent** updates documentation
- **Coordinator** orchestrates everything, escalates to you when uncertain

## Install

```bash
# 1. Clone this repo
git clone git@github.com:9aoyang/stackpilot.git ~/Documents/github/stackpilot

# 2. Install agent definitions globally
cp ~/Documents/github/stackpilot/agents/*.md ~/.claude/agents/

# 3. Initialize a project
cd /path/to/your/project
bash ~/Documents/github/stackpilot/scripts/init.sh
```

```bash

# If Coordinator needs your input:
# → You'll get a desktop notification
# → Edit tasks/NEEDS_REVIEW.md, add "REPLY: <your answer>"
# → Switch branches again to trigger Coordinator
